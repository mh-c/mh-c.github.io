Laravel Lifecycle Overview

Jun 5, 2017 00:00 · 170 words · 1 minute read

####First Things#### Entry point: public/index.php Key code: //loads the Composer generated autoloader definition require DIR.‘/../bootstrap/autoload.php’; //Retrieve an instance of the app/service container $app = require_once DIR.‘/../bootstrap/app.php’;

####HTTP/Console kernels#### 1. Request comes in kernel 2. Kernal defines an array of bootstrapers who: configure error handling, configure logging, detect the application environment, and perform other tasks that need to be done before the request is actually handled. 3. Kernal defines a list of middleware who: handle read and writing HTTP session, determine maintenance mode, verify CSRF token etc… 4. Request comes in and Response goes out

Service Providers config\app.php find the providers arrays Register all, then boot

Dispatch request Once the application has been bootstrapped and all service providers have been registered, the Request will be handed off to the router for dispatching. The router will dispatch the request to a route or controller, as well as run any route specific middleware.

####Focus On Service Provides#### 1. App instance created 2. Service providers registered 3. Request handed to bootstrap application app\providers directory
